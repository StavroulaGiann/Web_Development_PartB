# Web_Development_PartB

# README-part-b.md

Το Μέρος Α είναι ένα στατικό site υλοποιημένο κυρίως με HTML, CSS, κσι JavaScript, που στοχεύει στην εμφάνιση πληροφοριών για την e-learning πλατφόρμα, χωρίς να απαιτείται σύνδεση με backend ή βάση δεδομένων. Είναι προσβάσιμο μέσω του path /parta/.

Το Μέρος Β της εργασίας αφορά την υλοποίηση μιας **δυναμικής web εφαρμογής** e-learning, η οποία περιέχει μαθήματα και βιβλία σχετικά με διάφορους κλάδους της Πληροφορικής και υποστηρίζει την εγγραφή χρηστών στην πλατφόρμα και σε μαθήματα της επιλογής τους To frontend λειτουργεί δυναμικά και επικοινωνεί απευθείας με το backend, το οποίο υλοποιείται με Node.js και Express και είναι υπεύθυνο για τη διαχείριση όλων των δεδομένων. Η αποθήκευση των δεδομένων γίνεται σε MongoDB, η οποία περιλαμβάνει πληροφορίες για χρήστες, βιβλία, μαθήματα, εγγραφές χρηστών σε μαθήματα και αξιολογήσεις. Παράλληλα, έχει υλοποιηθεί ένας βασικός μηχανισμός αυθεντικοποίησης χρηστών, βασισμένος σε JSON Web Tokens (JWT). Για την την αρχικοποίηση της βάσης δεδομένων με βιβλία και μαθήματα έχουν δημιουργηθεί seed scripts.

---

## Γενική Αρχιτεκτονική

Η εφαρμογή αποτελείται από:

- **Client (Frontend)**
  Single Page Application (SPA), υπεύθυνη για το UI και την αλληλεπίδραση με τον χρήστη.

- **Backend (Server)**
  Node.js / Express εφαρμογή που παρέχει REST API endpoints και διαχειρίζεται τα δεδομένα.

- **Database**
  MongoDB, όπου αποθηκεύονται χρήστες, βιβλία, μαθήματα, εγγραφές και αξιολογήσεις.

Ο client και το backend είναι ανεξάρτητες εφαρμογές και επικοινωνούν αποκλειστικά μέσω HTTP requests.

---

## Διαθέσιμα API Endpoints

Το backend εκθέτει REST API endpoints, όπως:

**Health & Static**
- `GET /health`
- `GET /uploads/*`

**About**
- `GET /api/about`

**Authentication**
- `POST /api/auth/register`
- `POST /api/auth/login`

**Books**
- `GET /api/books`
- `GET /api/books/:id`

**Courses**
- `GET /api/courses`
- `GET /api/courses/:id`

**Enrollments**
- `POST /api/enrollments`
- `GET /api/enrollments/user/:userId`

**Reviews**
- `POST /api/reviews`
- `GET /api/reviews/course/:courseId`

**Users**
- `POST /api/users`

Τα endpoints επιστρέφουν δεδομένα σε μορφή **JSON**.

## Ροή Δεδομένων (Data Flow)

1. Ο χρήστης αλληλεπιδρά με το γραφικό περιβάλλον του client (UI),
   π.χ. επιλέγοντας βιβλία ή μαθήματα, πραγματοποιώντας σύνδεση
   ή υποβάλλοντας μια αξιολόγηση.

2. Ο client επεξεργάζεται την ενέργεια του χρήστη και δημιουργεί
   ένα HTTP αίτημα (GET, POST, PUT ή DELETE), το οποίο αποστέλλεται
   στο αντίστοιχο REST API endpoint του backend.

3. Το backend λαμβάνει το αίτημα και:
   - ελέγχει τα δεδομένα του αιτήματος
   - εκτελεί τη σχετική λογική
   - επικοινωνεί με τη βάση δεδομένων (MongoDB) για ανάγνωση ή αποθήκευση δεδομένων. 
     Η επικοινωνία γίνεται μέσω Mongoose, μιας βιβλιοθήκης που λειτουργεί ως ενδιάμεσος μεταξύ Node.js και MongoDB.

4. Αφού ολοκληρωθεί η επεξεργασία, το backend επιστρέφει την απάντηση
   στον client σε μορφή JSON, μαζί με τον κατάλληλο HTTP status code, (π.χ. 200, 201, 400).

5. Ο client λαμβάνει το JSON response, ενημερώνει την κατάσταση
   της εφαρμογής (state) και ανανεώνει δυναμικά το UI, χωρίς
   ανανέωση ολόκληρης της σελίδας, σύμφωνα με τη λογική μιας
   Single Page Application (SPA).

## Ελάχιστη Λειτουργικότητα που Υλοποιήθηκε

Στο πλαίσιο του Part B υλοποιήθηκαν τουλάχιστον τα εξής:

- **Δυναμική φόρτωση δεδομένων (books, courses)**  
  Τα δεδομένα βιβλίων και μαθημάτων δεν είναι στατικά, αλλά ανακτώνται
  δυναμικά από το backend μέσω REST API και εμφανίζονται στο UI του client.

- **Επικοινωνία client–backend μέσω REST API**  
  Ο client επικοινωνεί με το backend στέλνοντας HTTP αιτήματα (GET, POST)
  και λαμβάνει απαντήσεις σε μορφή JSON, σύμφωνα με την αρχιτεκτονική REST.

- **Single Page Application με client-side routing**  
  Η εφαρμογή λειτουργεί ως SPA, επιτρέποντας πλοήγηση μεταξύ σελίδων
  χωρίς ανανέωση ολόκληρης της σελίδας, βελτιώνοντας την εμπειρία χρήστη.
    
- **Εγγραφή χρηστών σε μαθήματα**  
  Οι χρήστες μπορούν να εγγραφούν σε μαθήματα και
  να αποθηκεύονται στη βάση δεδομένων μέσω του backend.

- **Υποβολή και προβολή αξιολογήσεων**  
  Υλοποιήθηκε η δυνατότητα υποβολής αξιολογήσεων για μαθήματα και
  η δυναμική προβολή τους στο UI.

- **Διαχωρισμός στατικού (Part A) και δυναμικού μέρους (Part B)**  
  Το στατικό μέρος της εφαρμογής (Part A) λειτουργεί ανεξάρτητα, ενώ το Part B αποτελεί τη δυναμική εφαρμογή
  που επικοινωνεί με το backend
